## 2026-03-28 i18n / Localization Pass

This pass is not visible in screenshots — the UI appearance is unchanged. What changed under the hood:

- Full i18n system wired: 25 locales, flat-string key architecture in `resources.ts` (~865 keys per locale)
- RTL layout active for Arabic (`ar`) and Hebrew (`he`) — `document.documentElement.dir` set automatically on language change
- Language switcher now covers all 25 supported locales with native-language names (Français, Español, العربية, 日本語, etc.)
- Language preference persisted to `localStorage` across reloads
- Page-level `t()` coverage added to: news, predictions, scout-workspace, team-fit, dashboard, rankings, tables, transfers, social, live-sim, match-rooms, odds, players, player profile, compare, fan-sentiment, games surfaces
- Shell nav and all tab labels, section headers, button labels, filter labels, and status badges are now translatable

Screenshots from the 2026-03-27 density/responsive/source passes remain the accurate visual reference for the current frontend. No visual regression from this pass.

Next screenshot capture should happen after the first real backend/data wiring pass or any significant visual surface change.
