# MOYOSOREJOBI Style

This is the writing rule for this repo.

Use this style for:

- commit messages
- PR comments
- inline code comments
- progress notes
- task summaries
- short docs added during work

## Core Style

- keep it direct
- keep it short
- say the point fast
- no fluff
- no fake polish
- no fake confidence
- no corporate tone
- no long explanations unless needed

## Comment Rule

- 1 to 15 words max
- one idea only
- action first when possible

Good:

- `fix mobile tab overflow`
- `keep this demo-only`
- `avoid fake live wording`
- `sync demo after ui pass`

Bad:

- `This section handles the rendering logic for all of the responsive navigation experiences in the application.`

## Commit Rule

- author name must be `MOYOSOREJOBI`
- keep commit messages brief
- 1 to 15 words max
- use lower-case unless a name needs caps
- describe the real change only

Good:

- `fix mobile loading trap`
- `tighten market xi spacing`
- `rewrite docs for current repo`
- `sync frontend demo`

Bad:

- `This commit contains a comprehensive set of updates to improve various parts of the user experience`

## Doc Rule

- keep docs clean
- keep docs readable
- keep sections tight
- use simple words first
- do not pad the writing
- do not make the repo sound more finished than it is

## Honesty Rule

- if something is mock, say it
- if something is broken, say it
- if something is not verified, say it

## Default Tone

- calm
- direct
- practical
- sharp
